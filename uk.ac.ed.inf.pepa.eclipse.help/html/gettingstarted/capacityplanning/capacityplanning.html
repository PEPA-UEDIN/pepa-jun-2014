This guide

This guide provides a step by step description of the Capacity Planning Tool (CPT).

It shows how to configure the tool in order to search for some optimal model configuration of system components. This page gives an overview of the tool in order to provide insight into how the tool works, and therefore how to configure the tool for a search. We start with a breif motivation, and then provide a high level view of the CPT.

Familiarity with PEPA is assumed. To learn more about PEPA visit the PEPA web site.

The Capacity Planning Tool

Motivation

Real systems are expected to service some demand with good performance. For example; a cloud based web-service may have a Service Level Agreement (SLA). This SLA might stipulate that the web-service will have a number of clients when operating (service demand) and must respond to all client requests within a certain time (performance).

Real systems are usually complex systems, and are compositions of resources. A naive approach to satisfying the SLA would be to continue adding resources to the system until the system starts performing as required. We want to avoid over-provisioning, where we have under-used resources, these are adding to our operational cost unnecessarily. We want to avoid under-provisioning, as this means our system is not going to meet the SLA.

We want an optimum configuration of resources, a configuration that uses the fewest number of resources, but still provides the expected service demand with a good performance. Searching for optimal configurations is time consuming and difficult, the CPT is an automated tool designed to aid the modeller in finding an optimal confguration, and in gaining some insight into the systems' behaviour when we consider the number of resources as a parameter.

Tool overview

  1. The designer creates a PEPA model in the PEPA editor.
  2. Then the designer starts the CPT choosing either a Driven Particle Swarm Optimisation (dPSO) or Single Particle Swarm Optimisation (PSO) search.
  3. The search engine creates ODE graphs in order to evaluate each model.
  4. The ODE evaluation passes back the measured performance.
  5. The CPT passes a tree view of the results, with the model configuration with the least cost at the top.
  6. The CPT also passes a list view of the results, sorted in ascending cost.

Performance evaluation

The CPT can evaluate models on:
  1. Capacity Utilisation
  2. Population levels
  3. Throughput
  4. Average Response-Time

Search algorithm

The CPT uses Particle Swarm Optimisation (PSO) to search through a model configuration search space. In a dPSO the CPT uses a Hill Climbing algorithm in order to find the best PSO settings for a model configuration search.
