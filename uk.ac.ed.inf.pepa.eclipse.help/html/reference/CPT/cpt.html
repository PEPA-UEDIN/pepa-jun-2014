Capacity Planning Tool

Component population

The number of components in a system. If we have:
Component[x]
x then is the population size of the resource in the system.

Component population range

The designer must specify an upper and lower limit to each component population size. This provides a component population range.

Model configuration search space

The set of all possible model configurations.

Model configuration

A model configuration is any model who has each of its component population sizes inside of their user defined component population ranges.

Particle Swarm Optimisation (PSO)

A PSO is used to search through the model configuration search space. A particle moves through the model configuration search space looking for an optimal model configuration based on a user defined cost function. The PSO in the CPT has five parameters: Original velocity, personal best, and global best effect the behaviour of the particles. The designer can set the number of particles searching using the Population count. Generation is the number of moves by a particle. Lastly the designer can set the number of times a PSO will run, the number of experiments.
see wiki for details.

PSO parameters

  1. Original velocity
  2. Personal best
  3. Global best
  4. Population count
  5. Generation
  6. Experiments

Driven Particle Swarm Optimisation (dPSO)

A dPSO uses a Hill Climbing algorithm to find the optimum Particle Swarm Optimisation (PSO) parameters. A Hill Climbing algorithm has far fewer parameters to set than a PSO. It therefore offers the designer an easier interface than the single PSO. However it takes longer to complete than a single PSO due to the additional search.

dPSO parameters

  1. Generation
  2. Experiments
